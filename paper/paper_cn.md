## 摘要

LLaDA 等离散掩码扩散语言模型通过迭代去噪生成文本：逐步将掩码 token 替换为预测 token。LLaDA2.1 引入了 Token-to-Token（T2T）编辑机制，通过直接替换疑似错误的已提交 token 来加速生成。然而，我们发现 T2T 编辑存在根本性局限：它将错误检测与替换耦合在一起，用可能错误的 token 污染生成上下文，并引入训练-推理噪声不匹配（系统性的模型生成错误与训练中见到的随机扰动截然不同）。

我们提出 **Token-to-Mask（T2M）重掩码**，一种无需训练、即插即用的 T2T 替代方案：将疑似错误 token 重置为掩码状态，让扩散过程在更干净的上下文下重新预测。我们设计并验证了三种互补的错误检测策略——基于概率、基于触发镜像、基于时序差分——并提供统一理论分析，表明 T2M 能净化上下文、将系统性推理错误转回模型原生的掩码噪声类型、并实现延迟提交以进行多位置联合优化。

在涵盖知识、推理、数学、代码和指令遵循的 9 个基准上的全面实验表明，T2M 在大多数基准上优于原始 T2T 编辑基线，在需要精确 token 级输出的任务上增益最大。在 CMATH 上的错误分析揭示，主导失败模式是**末端 token 损坏**——模型推理正确但最终答案在去噪过程中被损坏——T2M 修复了 67.6% 的此类情况。

## 1 引言

自回归（AR）语言模型主导了文本生成，但其从左到右的顺序解码本质上是串行的。离散掩码扩散语言模型（dLLM）提供了一种引人注目的替代方案：通过迭代去噪一个全掩码序列来生成文本，每个去噪步骤中可以并行预测 token。最近的 dLLM 如 LLaDA 和 Dream 已在广泛基准上取得了与同规模 AR 模型相当的性能。

dLLM 通过并行解码实现了显著的推理加速，但这种并行性也有代价。在每个去噪步骤中，token 预测是条件独立的——每个位置的采样不依赖于同时生成的邻居——这系统性地放大了 token 级的不一致性。这种更高的错误易感性，加上模型基于自身不完美预测的曝光偏差（exposure bias），使得错误纠正成为生成质量的关键瓶颈。

为此，LLaDA2.1 引入了一项关键的推理时创新：**Token-to-Token（T2T）编辑**。除了标准的 Mask-to-Token（M2T）步骤填充掩码位置外，T2T 在模型以高置信度预测不同 token 时直接替换已提交的（非掩码）token。凭借这些创新，LLaDA2.1 已展示了在同等规模下与领先 AR 模型相当的性能，确立了 dLLM 作为实用文本生成替代方案的地位。在不增加训练成本的前提下提升这样一个系统的推理质量，具有直接的实际意义。

然而，我们发现 T2T 编辑存在三个根本性局限：

### 局限 1：检测-替换耦合

T2T 将"检测错误"和"提供替换"耦合在了同一个触发条件中：只有当模型对某个**不同于**当前 token 的候选预测概率超过阈值 τ_t2t 时才会触发（即 p_θ(x_i*|**z**) > τ_t2t 且 x_i* ≠ x_i^old）。换言之，T2T 无法单独表达"这个 token 不对"而不同时给出"应该换成什么"。当条件分布是多峰的（多个合理 token 分散了概率），即使模型明确认为当前 token 不对，也可能没有任何单一候选超过阈值，导致错误 token 被留在原位——我们称之为**纠正惰性**（correction inertia）（见图 1a）。

### 局限 2：上下文污染（T2T 的根本性问题）

T2T 的核心设计——用一个 token 替换另一个——无法保证上下文改善。即使 T2T 被触发，其替换是在可能本身包含错误的上下文下条件化的；在这种被污染的上下文下，即使高置信度的替换也可能是错误的。被替换的 token 作为看似合法的信号，提供误导性语义线索，偏置所有其他位置的预测，并可能形成自我强化的错误循环（见图 1b、图 2）。

### 局限 3：训练-推理噪声不匹配（LLaDA2.1 训练流程的问题）

LLaDA2.1 的 T2T 训练流使用**随机** token 扰动（均匀采样的错误 token → 正确答案），但推理时的错误 token 是**系统性、模型生成的**错误，它们语义上合理且与周围 token 相关。这意味着模型在推理时遇到的上下文噪声是训练中从未见过的第三种类型——既不是 M2T 流的干净掩码噪声，也不是 T2T 流的随机扰动，而是系统性的、语义连贯的模型自身错误（见图 2 中 adversarial context 的实证验证）。

### 我们的方案：Token-to-Mask 重掩码

我们提出 T2M 重掩码作为原则性的替代方案。T2M 不替换疑似 token，而是将其重置为 [M] 状态，让扩散过程在更干净的上下文下从头重新预测。这个简单的改变同时解决了三个局限：解耦错误检测与替换、通过移除错误 token 净化上下文、将系统性推理错误转回模型原生的掩码噪声类型。

此外，通过回退到 [M] 而非立即提交替换，T2M 自然实现了**延迟提交**（delayed commitment）——将最终 token 决策推迟到周围上下文进一步收敛的后续步骤，实现多个可疑位置的联合重优化（见图 1c）。

我们的方法**无需训练**——不需要额外训练或架构修改，可作为 LLaDA2.1 现有生成流水线中 T2T 的即插即用替代。

### 贡献

1. 我们发现 LLaDA2.1 中 T2T 编辑的根本性局限，提出 T2M 重掩码作为原则性的、无需训练的替代方案。
2. 我们设计并验证了三种错误检测策略，分析其互补优势。
3. 我们提供统一理论框架，从训练-推理噪声不匹配、上下文信号层次和延迟提交的角度解释为什么掩码 token 严格优于错误 token 作为上下文。
4. 我们发现**末端 token 损坏**是主导错误模式（CMATH 上 73.9% 的错误），并表明 T2M 专门修复此失败模式。
5. 我们在 9 个基准上进行全面实验，在需要精确 token 级输出的任务上展示了广泛改进。

## 2 背景

### 2.1 掩码扩散语言模型

掩码扩散语言模型定义了一个前向加噪过程（逐步掩码 token）和一个反向去噪过程（恢复 token）。给定干净序列 **x** = (x₁, ..., x_L)，前向过程在时间 t ∈ [0,1] 以概率 α_t 独立地将每个 token 替换为掩码 [M]。

模型 p_θ(**x** | **z**_t) 通过交叉熵损失训练，同时预测所有被掩码位置。

推理时，从全掩码序列开始迭代：(1) 预测所有掩码位置，(2) 保留置信度超过阈值的预测，受限于由解掩码调度确定的预算上限，直到所有掩码被填充。

### 2.2 块扩散

LLaDA2.1 采用**半自回归**块扩散架构。回复被分为 B=32 个 token 的块，从左到右**顺序**生成；每个块内，token 通过迭代去噪**并行**生成。

- **提示与回复处理**：提示 token 不掩码放入序列，只有回复位置初始化为 [M]。
- **块因果注意力**：块内双向注意力，块间严格因果（块 j 只能关注块 0..j）。已生成块冻结，KV 状态缓存复用。
- **迭代 M2T 去噪**：每次迭代预测所有掩码位置，置信度超过 τ_m2t 的被提交；不足预算时取 top-k。

### 2.3 Token-to-Token (T2T) 编辑

除 M2T 外，LLaDA2.1 在每个去噪步骤引入 T2T 编辑阶段。对已包含非掩码 token 的位置，如果模型以超过 τ_t2t 的置信度预测了不同 token，则直接替换：

> x_i ← x_i* 当 p_θ(x_i* | **z**) > τ_t2t 且 x_i* ≠ x_i^old

内循环持续到所有掩码被填充且没有进一步 T2T 编辑被触发。

## 3 方法：Token-to-Mask 重掩码

### 3.1 概览

我们用 T2M 重掩码替换 T2T 编辑。当错误检测机制识别出一个已提交 token 可能不正确时，将其重置为 [M]：

> x_i ← [M] 当 ShouldRemask(i, **z**, θ)

后续 M2T 步骤在更新的上下文下重新预测该位置。这是对 LLaDA2.1 生成循环的最小手术式改动：只有 token 编辑阶段被替换；所有其他组件保持不变。

### 3.2 错误检测策略

**策略 1：LowProb——当前概率低**

模型在当前上下文下重新评估所有已提交 token。如果分配给当前 token 的概率低于 τ_lp，则重掩码：

> Remask_LP(i) = 𝟙[p_θ(x_i^old | **z**_{-i}) < τ_lp]

直接测试"模型是否仍然认为这个 token 属于这里？"——不需要特定的替代候选。

**策略 2：T2T-Remask——相同触发，不同动作**

使用与 T2T 完全相同的触发条件（模型高置信度预测了不同 token），但执行重掩码而非替换：

> Remask_T2T(i) = 𝟙[p_θ(x_i* | **z**) > τ_tr ∧ x_i* ≠ x_i^old]

通过保留检测逻辑但改变动作，隔离了基于掩码的纠正与基于 token 的替换的效果差异。

**策略 3：LogitDiff——置信度下降**

比较同一块内两个连续去噪迭代的模型输出。当模型对某个 token 的置信度下降幅度超过阈值 τ_ld 时重掩码：

> Remask_LD(i) = 𝟙[p_θ^(t-1)(x_i^old | **z**^(t-1)) - p_θ^(t)(x_i^old | **z**^(t)) > τ_ld]

置信度下降表明随着周围位置被填充，演变的上下文不再支持该 token。与 LowProb 使用单步快照不同，LogitDiff 捕获跨迭代的**动态**上下文演变。

### 3.3 安全机制

- **单位置重掩码预算**：每个位置维护计数器 c_i，重掩码时递增。c_i ≥ C_max 时阻止重掩码（默认 C_max=1）。
- **重掩码比例上限**：每步最多 ρ_max（默认 0.25）比例的可编辑位置可被重掩码。超出时优先选择置信度最低的。

### 算法伪代码

```
输入：当前块 token z, 模型 p_θ, 策略 S, 阈值 τ, 预算 C_max, 比例上限 ρ_max

// 标准 M2T 步骤（与 LLaDA2.1 相同）
对每个掩码位置 i (z_i = [M]):
    如果 max_v p_θ(v|z) > τ_m2t:
        z_i ← argmax_v p_θ(v|z)

// T2M 重掩码步骤（替换 T2T 编辑）
E ← {i : z_i ≠ [M] 且 i ∉ prompt}          // 可编辑位置
R ← {i ∈ E : ShouldRemask(i,z,τ) 且 c_i < C_max}
如果 |R| > ρ_max · |E|:
    R ← 按最低置信度取 top-k, k = ⌊ρ_max · |E|⌋
对每个 i ∈ R:
    z_i ← [M]; c_i ← c_i + 1
返回 R = ∅                                    // 无重掩码则已收敛
```

## 4 理论分析

### 4.1 解耦错误检测与纠正

T2T 将两个逻辑独立的决策耦合为单一条件：(1) 当前 token 是否错误？(2) 是否有高置信度替代？这种耦合产生**纠正惰性**：尽管有明确证据表明 token 错误，系统仍无法发起纠正，因为没有单一替代达到置信度阈值。

我们通过**卡住集合**（stuck set）形式化：

> S_stuck = {i : p_θ(x_i^old | **z**_{-i}) < ε, 但 max_v p_θ(v | **z**_{-i}) < τ_t2t}

当条件分布多峰时，这种情况自然出现。例如，用 LLaDA2.1-mini 补全 "He felt very ____."：候选 token "sad"（0.12）、"happy"（0.11）、"excited"（0.05）各有一定概率，而错误的已提交 token "purple" 概率仅 ~0.00002。T2T 在 τ_t2t=0.5 时不采取任何行动（因为 max_v p_θ(v) = 0.12 < 0.5），尽管模型明显识别了错误（见图 1a）。

由此可见，T2T 在 S_stuck 上完全失效：触发条件要求 max_v p_θ(v|z_{-i}) > τ_t2t，而 S_stuck 的定义恰好排除了这一点。相比之下，LowProb 仅检查 p_θ(x_i^old|z_{-i}) < τ_lp，不要求模型对任何替代 token 有高置信度。由于 S_stuck 中 p_θ(x_i^old) < ε ≪ τ_lp，LowProb 必然触发重掩码，将位置交由后续 M2T 步骤在更收敛的上下文下重新预测。需注意，重掩码是发起纠正的必要步骤，但不保证后续一定预测正确——关键在于 T2T 连这一步都无法执行。

### 4.2 错误上下文净化

T2M 最关键的优势在于重新预测时的**上下文质量**。

**上下文信号层次**（核心洞察）：

对于预测位置 i，每个上下文位置 j ≠ i 提供三种信号之一：
1. **对齐信号**（正确 token）：将预测偏向正确答案 ✓
2. **空信号**（[M]）：不向任何方向偏置预测 ○
3. **对抗信号**（错误 token）：将预测偏**离**正确答案 ✗

在此层次下，T2M 将类型(3)替换为类型(2)，零代价消除对抗偏差。T2T 将类型(3)替换为可能仍是类型(3)的新 token。

**实证验证**："I went to [X] and visited the [M] Tower."
- [X]=France → p(Eiffel) = 0.97（对齐 ✓）
- [X]=[M] → p(Eiffel) = 0.82（空 ○）
- [X]=Japan → p(Tokyo) = 0.91（对抗 ✗，高置信度的错误答案！）

这证明对抗信号严格劣于空信号，T2M 的"对抗→空"转换总是安全的。

**错误传播循环**：当多个对抗信号共存时，位置 j 的错误偏置位置 i 的预测，反过来强化 j 的错误，形成稳定但错误的不动点。T2M 通过同时将多个对抗信号转为空信号来打破此循环。

### 4.3 训练-推理噪声不匹配

LLaDA2.1 采用双流训练：M2T 流从掩码上下文预测，T2T 流从随机扰动位置恢复正确 token。但训练中的扰动 token 是**均匀随机**采样的（与上下文无关、语义不连贯），而推理时的错误 token 是**系统性、模型生成的**（语义合理、局部连贯、与周围 token 相关）。

将错误转为 [M] 后：
> {z_correct, z_error} →T2M→ {z_correct, [M]}

T2M 用中性的"未知"标记替换主动误导的信号，利用模型最可靠的操作模式。

**连续扩散类比**：在连续扩散模型中，如果测试时的噪声分布偏离训练分布（如结构化对抗扰动代替随机高斯噪声），分数估计会退化。类似地，在掩码扩散中，推理时的系统性错误构成第三种未见过的噪声类型。T2M 将其转回熟悉的 [M] 类型。

### 4.4 延迟提交与联合优化

T2T 是**贪心**策略：一旦发现高置信度替代就立即提交。这有两个问题：过早锁定（替换可能次优）和逐位置独立优化（忽略位置间依赖）。

T2M 实现**延迟提交**：通过将可疑位置重置为 [M]，推迟到更多上下文收敛后的步骤再做最终决策。当多个位置同时被重掩码时，后续 M2T 步骤有效地进行**联合**优化——上下文**整体更干净**（所有错误同时被移除），产生更好、更全局一致的预测。

这类似于**模拟退火**：T2M 局部增加不确定性来逃离局部最优，探索更全局最优的解。

**示例**：图 1b 展示了过早锁定的问题（DROP，正确答案=857）。在早期去噪步骤中，当只有 "8"（p=0.11）已提交而 "5" 和 "7" 还是 [M] 时，T2T 基于不完整上下文贪心地将 "8" 替换为 "6"（p=0.64 > τ_t2t），产生错误答案 657。T2M 则重掩码 "8"，等待 "5" 和 "7" 被提交后，在完全收敛的上下文下重新预测 "8"（p=0.94）。

图 1c 展示了多位置联合优化（DROP，正确答案="Jon Kitna"）。T2T 贪心地将 "Jon" 替换为 "Kit"（p=0.57），破坏了名字。T2M 迭代重掩码不确定 token（6 轮），让 "Jon" 保持稳定，而 "Kit" 和 "na" 逐步收敛到 p≥0.98——系统通过跨多个相互依赖位置的延迟提交找到了全局一致的解。

## 5 相关工作

### 离散掩码扩散语言模型

D3PM 引入了离散吸收态扩散。MDLM 和 SEDD 开发了高效训练目标。LLaDA 扩展到 8B 参数，LLaDA2.1 引入 T2T 编辑。Dream 进一步证明了扩散语言模型在规模上的可行性。

### 掩码扩散中的重掩码

**ReMDM** 是重掩码的开创性工作，修改反向后验引入概率重掩码参数 σ_t，允许任何已解掩码 token 在每步被随机重掩码。ReMDM 在**采样过程级别**操作。相比之下，我们在 LLaDA2.1 的**编辑步骤级别**操作，基于显式错误检测进行**定向**重掩码。

我们形式化证明定向重掩码严格优于随机重掩码：
- 随机重掩码：Q_random(σ) = (1-σ)(N_c·s₊ + N_e·s₋)——增加 σ 在移除对抗信号的同时等量破坏对齐信号
- 定向重掩码：Q_targeted = N_c·s₊——只移除错误，保留所有正确
- 优势 Q_targeted - Q_random > 0 对所有 σ ∈ (0,1] 成立

**CORE** 通过掩码扰动的敏感性来识别"上下文脆弱"token。我们指出其敏感性指标可能受去噪过程的瞬态掩码模式影响，而非 token 正确性本身。静态概率 p_θ(x_i|**z**) 是给定完整上下文的模型后验信念，由数据处理不等式可知，在人为退化上下文下评估不能产生比全上下文预测更多的信息。

**基于训练的方法**（RemeDi、ProSeCo、PRISM、MDPO）通过额外训练改进重掩码质量。它们与我们互补：它们改进模型检测/纠正错误的能力，我们改进纠正**机制**本身（T2M vs T2T）。

## 6 实验

### 6.1 实验设置

- **模型**：LLaDA2.1-mini（16B 参数，MoE），相同预训练权重，无额外训练
- **推理参数**：Q Mode 默认——τ_m2t=0.7, τ_t2t=0.5, 块长度 B=32, 温度=0.0（贪心）
- **重掩码参数**：LowProb 策略（τ=0.3, C_max=1, ρ_max=0.25），从消融实验中选择的最具成本效益的配置
- **评估说明**：使用官方模型权重和数据集。由于 LLaDA2.1 未开源评估代码，我们复现的基线绝对分数可能与原文有差异；所有 T2T 与 T2M 的对比在完全相同的评估流水线下进行
- **基准**：9 个基准，覆盖知识（GPQA、TriviaQA）、推理（HellaSwag、PIQA、DROP）、数学（CMATH、AIME 2025）、代码（HumanEval+）、指令遵循（IFEval）

### 6.2 主要结果

| 类别 | 基准 | Original (T2T) | T2M (LowProb) | Δ |
|------|------|:--------------:|:--------------:|:---:|
| 知识 | GPQA-Diamond | 21.7 | **22.7** | +1.0 |
| 知识 | TriviaQA (F1) | 30.7 | **31.9** | +1.2 |
| 推理 | HellaSwag | **81.3** | 81.2 | -0.1 |
| 推理 | PIQA | **79.2** | 79.1 | -0.1 |
| 推理 | DROP (F1) | 54.7 | **55.3** | +0.6 |
| 数学 | CMATH | 82.9 | **88.7** | **+5.8** |
| 数学 | AIME 2025 | 33.3 | **40.0** | **+6.7**† |
| 代码 | HumanEval+ | 64.6 | 64.6 | ±0.0 |
| 指令 | IFEval (Strict) | 73.0 | **73.4** | +0.4 |

†AIME 2025 仅 30 题，+6.7% 对应多答对 2 题，样本量小需谨慎解读。

**关键发现**：
- **数学任务增益最大**（CMATH +5.8, AIME +6.7）：最终答案是精确数字，单个数位损坏就导致错误
- **知识与抽取式 QA 有改善**（GPQA +1.0, TriviaQA +1.2 F1, DROP +0.6 F1）：答案是需要精确复现的短文本
- **常识推理基本不变**（HellaSwag -0.1, PIQA -0.1）：输出短（~100-140 tokens），很少出现损坏
- **代码无差异**（HumanEval+ ±0.0）

### 6.3 消融实验

在 CMATH 上进行全交叉消融（100 样本），共 109 种配置：

| 策略 | 更高 τ → | 扫描范围 |
|------|---------|---------|
| LowProb | 更多重掩码 | {0.1, 0.3, 0.5, 0.7, 0.9} |
| T2T-Remask | 更少重掩码 | {0.5, 0.7, 0.9} |
| LogitDiff | 更少重掩码 | {0.1, 0.2, 0.3, 0.5} |

主要趋势：
- **所有策略均优于 T2T**：每个 T2M 配置都超过 T2T 基线（81%）
- **LowProb 主导**：各成本级别下准确率最高。τ=0.3 以仅 69 次重掩码达到 93%
- **T2T-Remask 最廉价**：τ=0.9 仅触发 ~2 次重掩码/样本就达到 87%（+6%）
- **过度重掩码有害**：>1000 次重掩码/样本时准确率降至 88% 以下

最具成本效益的配置（LowProb τ=0.3, C=1, ρ=0.25）仅增加 ~69 次重掩码/样本（远小于平均输出长度 ~333 tokens），即获得 12 个百分点的准确率提升。

### 6.4 分析

**逐样本翻转分析**：

| 基准 | N | 双对 | 双错 | 仅T2T对 | 仅T2M对 | 净增 |
|------|--:|-----:|-----:|--------:|--------:|-----:|
| CMATH | 1098 | 869 (79.1%) | 83 (7.6%) | 41 (3.7%) | **105** (9.6%) | +64 |
| AIME 2025 | 30 | 9 (30.0%) | 17 (56.7%) | 1 (3.3%) | **3** (10.0%) | +2 |
| DROP | 9535 | 4150 (43.5%) | 5004 (52.5%) | 144 (1.5%) | **237** (2.5%) | +93 |
| TriviaQA | 17944 | 2488 (13.9%) | 15206 (84.7%) | 57 (0.3%) | **193** (1.1%) | +136 |
| GPQA | 198 | 25 (12.6%) | 135 (68.2%) | 18 (9.1%) | **20** (10.1%) | +2 |
| HellaSwag | 10042 | 7974 (79.4%) | 1696 (16.9%) | 194 (1.9%) | 178 (1.8%) | -16 |
| PIQA | 1838 | 1412 (76.8%) | 341 (18.6%) | 44 (2.4%) | 41 (2.2%) | -3 |

**末端 token 损坏：主导错误模式**

在 CMATH 上的系统性错误分析中，T2T 的 188 个错误中**139 个（73.9%）是末端 token 损坏**：模型的 CoT 推理得出正确答案，但最终提取的答案在去噪过程中被损坏。仅 49 个（26.1%）是真正的推理失败。

损坏的典型形式：
- **追加多余数字**（18 → 188，43 例）
- **丢失数字**（32 → 2，25 例）
- **答案标记重复**（24 例）
- **单数字替换**（14 例）

| | 总错误 | 损坏 | 真正推理错误 | 准确率 |
|---|:---:|:---:|:---:|:---:|
| T2T 编辑 | 188 | 139 (73.9%) | 49 (26.1%) | 82.9% |
| T2M 重掩码 | 124 | 73 (58.9%) | 51 (41.1%) | 88.7% |
| Δ | -64 | -66 (↓47.5%) | +2 | +5.8% |

T2M 修复了 **94/139** 个损坏案例（67.6%），而真正推理错误几乎不变（49→51）。

**任务依赖性**：T2M 在需要**精确 token 级输出**的任务上帮助最大（数学、抽取式 QA）。在常识推理（短输出）和代码（需要结构正确性而非精确 token 保真度）上无明显收益。

---

## 7 结论

我们发现了 LLaDA2.1 中 T2T 编辑的根本性局限，提出 T2M 重掩码作为原则性的、无需训练的替代方案。T2M 通过将疑似错误 token 重置为掩码状态来净化生成上下文，将系统性推理错误转回模型原生的掩码噪声类型，并通过延迟提交实现多位置联合重优化。

在 9 个基准上的实验展示了广泛改进，数学任务增益最大。错误分析揭示末端 token 损坏占 CMATH 错误的 73.9%，T2M 修复了其中 67.6%。

**局限性**：方法专用于 LLaDA2.1 的 T2T 编辑机制；当错误率极低时掩码上下文未必优于错误上下文；安全机制的超参数可能需要任务特定调优。

**未来方向**：噪声感知继续训练（用真实模型生成的错误替代随机扰动来训练）；结合 T2M 与采样级重掩码（如 ReMDM）；自适应学习重掩码阈值（如 PRISM 框架）。
