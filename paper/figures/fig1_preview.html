Figure 1: T2T editing vs. T2M remasking. (a) T2T detects the erroneous token "purple" but cannot correct it: the conditional distribution is multimodal ("happy," "glad," "pleased" each ~0.25) so no single candidate exceeds the confidence threshold — a failure mode we call correction inertia. The error persists and pollutes context for subsequent predictions. (b) T2M detects the same error via low posterior probability and resets it to [M] (context purification). A subsequent M2T step re-predicts the position under cleaner context, producing the correct token (delayed commitment).
