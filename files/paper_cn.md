## 摘要

LLaDA 等离散掩码扩散语言模型通过迭代去噪生成文本——逐步将 mask token 替换为预测的 token。LLaDA2.1 引入了 Token-to-Token (T2T) 编辑机制，通过直接替换疑似错误的已提交 token 来加速生成。然而，我们发现 T2T 编辑存在根本性的局限：它将错误检测与替换耦合在一起，用可能不正确的 token 污染生成上下文，并存在训练-推理噪声不匹配（推理时系统性的模型生成错误与训练时的随机扰动本质不同）。我们提出 **Token-to-Mask (T2M) 重掩码**，一种 training-free 的、即插即用的 T2T 替代方案——将疑似错误 token 重置为 mask 状态，让扩散过程在更干净的上下文下重新预测。我们设计并在小规模验证集上初步验证了三种互补的错误检测策略（基于概率的、触发镜像的、时间差分的），并提供统一的理论分析，证明 T2M 重掩码能够净化生成上下文、将系统性推理错误转回模型原生 [M] 噪声类型、并通过延迟提交实现多位置联合优化。在跨越知识、推理、数学和编程的 XX 个 benchmark 上的全面实验表明，T2M 相比原始 T2T 编辑取得了一致的提升。

## 1. 引言

自回归 (AR) 语言模型主导了文本生成，但其从左到右的顺序解码本质上是串行的。离散掩码扩散语言模型 (dLLM) 提供了一种有吸引力的替代方案：它们通过迭代去噪一个全 mask 序列来生成文本，在每个去噪步骤内实现并行 token 预测。近期的 dLLM 如 LLaDA 和 Dream 已经在广泛的 benchmark 上达到了与同规模 AR 模型竞争的性能。

虽然 dLLM 通过并行解码实现了显著的推理加速，但这种并行性也付出了代价：每个去噪步骤内 token 预测的条件独立性——每个位置在采样时没有对同时生成的邻居的顺序因果条件——系统性地放大了 token 级别的不一致性，使得 dLLM 的 token 错误率高于 AR 模型 (Kang et al., 2025)。这种更高的错误易感性，加上模型对自身不完美预测的暴露偏差 (exposure bias) 的复合效应，使得错误纠正成为生成质量的关键瓶颈。

为了解决这一问题，LLaDA2.1 引入了一个关键的推理时创新：**Token-to-Token (T2T) 编辑**。除了标准的 Mask-to-Token (M2T) 步骤（填充 mask 位置）之外，T2T 在模型对非 mask 位置以高置信度预测出不同 token 时，直接替换该位置已提交的 token。这一机制针对错误纠正瓶颈，使模型能够回溯性地"改变主意"。

然而，我们发现 T2T 编辑存在**三个根本性的局限**：

1. **检测-替换耦合。** T2T 要求同时满足两个条件：当前 token 错误，*并且*存在一个高置信度的替代。当正确分布是多模态的（多个合理 token）时，没有单个候选能超过置信阈值，导致明显错误的 token 无法被纠正——我们将这种失败模式称为**纠正惰性 (correction inertia)**。

2. **上下文污染。** 在并行生成中，错误 token 不可避免地出现在上下文中——来自不准确的 M2T 预测或不正确的 T2T 替换。这些错误 token 以合法 token 的形式出现，提供误导性的语义信号，偏置了 mask 和非 mask 位置上的后续预测。T2T 在不存在高置信度替代时不会被触发，错误因此原地保留；即使触发，当 T2T 尝试纠正检测到的错误时，其替换本身也可能不正确，延续了上下文污染。T2M 则通过重置为中性 [M] 彻底移除错误信号。

3. **训练-推理噪声不匹配。** LLaDA2.1 的 T2T 训练使用随机 token 扰动（随机错误 token → ground truth），但推理时上下文中的错误 token 不是随机扰动——它们是**系统性的、相关的**模型生成错误。这种分布差异同时影响 M2T 和 T2T 操作：无论操作类型如何，推理时遇到的上下文噪声与训练时见到的随机扰动根本不同。由于 M2T 从 [M] 上下文预测是掩码扩散模型的基础能力，将错误重置为 [M] 可将上下文转换为模型处理最可靠的形式。

我们提出 **Token-to-Mask (T2M) 重掩码**作为有原则的替代方案。不是替换可疑 token，而是将其重置为 [M] 状态，让扩散过程在更干净的上下文下从头重新预测这些位置。这个简单的改变同时解决了上述三个问题：它解耦了错误检测和替换、通过移除错误 token 净化了上下文、并将系统性推理错误转换回模型原生的 [M] 噪声类型。此外，通过退回 [M] 而非立即提交替换 token，T2M 天然实现了**延迟提交 (delayed commitment)**——将最终的 token 决策推迟到周围上下文进一步收敛的后续步骤，从而实现多个可疑位置的联合重优化。图 1 直观展示了这一对比：T2T 在无高置信替代时将错误原地保留（*纠正惰性*），而 T2M 将其重置为 [M]，让后续 M2T 步骤在净化的上下文下恢复正确 token。

我们的方法是 **training-free** 的——不需要额外训练或架构修改，作为 LLaDA2.1 现有生成流水线中 T2T 的即插即用替代。我们设计并在小规模验证集上初步验证了三种互补的错误检测策略（LowProb、T2T-Remask 和 LogitDiff），每种针对 token 不可靠的不同信号。我们从四个视角提供了统一的理论分析：解耦检测、上下文净化、训练-推理噪声不匹配和延迟提交。

**我们的贡献：**
- 发现 LLaDA2.1 中 T2T 编辑的根本性局限，提出 T2M 重掩码作为有原则的、training-free 的替代
- 设计并在小规模验证集上初步验证三种错误检测策略，分析其互补优势
- 提供统一的理论框架，从训练-推理噪声不匹配、上下文信号层级和延迟提交的角度解释为什么 mask 严格优于错误 token 作为上下文
- 在 XX 个 benchmark 上进行全面实验，验证一致的提升

## 2. 背景

### 2.1 掩码扩散语言模型

掩码扩散语言模型定义了一个前向加噪过程（逐步掩码 token）和反向去噪过程（恢复它们）。给定干净序列 **x** = (x₁, ..., x_L)，前向过程在时间 t ∈ [0,1] 以概率 α_t 独立地将每个 token 替换为 mask token [M]（其中 α₀=0, α₁=1），得到噪声序列 **z**_t。

模型 p_θ(**x** | **z**_t) 被训练为同时预测所有 mask 位置，使用交叉熵损失：

$$\mathcal{L}(\theta) = -\mathbb{E}_{\mathbf{x}, t}\left[\frac{1}{\sum_i \mathbb{1}[z_t^i = \text{[M]}]} \sum_{i=1}^{L} \mathbb{1}[z_t^i = \text{[M]}] \log p_\theta(x_i \mid \mathbf{z}_t)\right]$$

推理时，从全 mask 序列开始，迭代地：(1) 预测所有 mask 位置，(2) 保留置信度超过阈值的预测，受 unmasking schedule 决定的预算上限约束——当满足阈值的位置超过预算时，仅提交置信度最高的 top-k 个，其余重新 mask。

### 2.2 Block Diffusion

LLaDA2.1 采用**半自回归 (semi-autoregressive)** 的 block diffusion 架构。响应被划分为大小为 B 个 token 的 block（通常 B=32），block 之间**从左到右顺序**生成；每个 block 内部的 token 通过迭代去噪**并行**生成。

**Prompt 与响应的处理。** 给定 prompt-response 对，prompt token 以**非掩码**形式放入序列——只有响应位置被初始化为 [M]。对于与 prompt 部分重叠的 block，prompt 位置被标记为不可编辑，在整个生成过程中保持冻结。

**Block-causal 注意力。** 为在保持 block 内并行生成的同时维持从左到右的连贯性，LLaDA2.1 采用 **block-causal** 注意力掩码。**Block 内部**，所有位置双向互相注意，使模型能够捕捉 block 内依赖关系。**Block 之间**，注意力严格因果：block j 可以注意 block 0, 1, ..., j，但不能注意任何未来 block j' > j。该 block 级下三角掩码在 block 级构建为 M_attn = tril(1_{N_b × N_b})，然后扩展到 token 级分辨率。已生成的 block 被冻结，其 key-value 状态被缓存以供后续 block 高效复用。

**迭代 M2T 去噪。** 每个 block 内，模型通过迭代的 Mask-to-Token (M2T) 步骤填充 [M] 位置。每次迭代中，模型对所有掩码位置预测 token 分布 p_θ(x_i | **z**)。置信度超过阈值 τ_m2t 的位置被提交；当满足阈值的位置数不足每步预算 n_transfer 时，改为提交置信度最高的 top-n_transfer 个预测。内循环重复直到 block 中所有 mask 被填满。

### 2.3 Token-to-Token (T2T) 编辑

除 M2T 外，LLaDA2.1 在每个去噪步引入了 **Token-to-Token (T2T)** 编辑阶段。对已包含已提交（非 mask）token 的位置，模型检查是否以超过 τ_t2t 的置信度预测了一个*不同*的 token。如果是，已提交 token 被**直接替换**：

$$x_i \leftarrow x_i^{*} \quad \text{若} \quad p_\theta(x_i^{*} \mid \mathbf{z}) > \tau_{\text{t2t}} \;\land\; x_i^{*} \neq x_i^{\text{old}}$$

其中 x_i* = argmax_v p_θ(v | **z**) 是模型的最高预测，x_i^old 是当前已提交的 token。Block 内的 prompt 位置不参与 T2T 编辑。

内循环持续到所有 mask 被填满且没有进一步的 T2T 编辑被触发。然后生成进入下一个 block，当前 block 的 token 被冻结作为上下文。

## 3. 方法：Token-to-Mask 重掩码

### 3.1 概述

我们用 **Token-to-Mask (T2M) 重掩码**替代 T2T 编辑。当错误检测机制识别出一个已提交 token 可能不正确时，不是用另一个 token 替换它，而是将其重置为 [M]：

$$x_i \leftarrow \text{[M]} \quad \text{若} \quad \text{ShouldRemask}(i, \mathbf{z}, \theta)$$

后续的 M2T 步骤会在更新的上下文下重新预测这个位置。这是对 LLaDA2.1 生成循环的最小、精准修改：只有 token 编辑阶段被替换（见算法 1）；所有其他组件（模型架构、M2T 逻辑、block 调度、KV 缓存）保持不变。

### 3.2 错误检测策略

我们为 ShouldRemask 设计并初步验证了三种互补策略，每种捕捉 token 不可靠的不同信号。

**策略1：LowProb — 当前概率低**

模型在当前上下文下重新评估所有已提交 token。如果分配给当前已提交 token 的概率低，则重掩码：

$$\text{Remask}_{\text{LP}}(i) = \mathbb{1}\left[p_\theta(x_i^{\text{old}} \mid \mathbf{z}_{-i}) < \tau_{\text{lp}}\right]$$

这个策略直接测试"模型是否仍然认为这个 token 属于这里？"而不需要特定的替代候选。

**策略2：T2T-Remask — 相同触发，不同动作**

这个策略使用与 T2T 相同的触发条件（模型自信地预测了一个*不同的* token），但执行重掩码而非替换：

$$\text{Remask}_{\text{T2T}}(i) = \mathbb{1}\left[p_\theta(x_i^{*} \mid \mathbf{z}) > \tau_{\text{tr}} \;\land\; x_i^{*} \neq x_i^{\text{old}}\right]$$

通过保持检测逻辑但改变动作，这个策略隔离了基于 mask 的纠正与基于 token 的替换之间的效果。

**策略3：LogitDiff — 置信度下降**

该策略比较同一 block 内**连续两次去噪迭代**的模型输出。设 p_θ^(t)(· | **z**^(t)) 为第 t 次迭代的模型预测，其中上下文 **z**^(t) 反映第 t-1 次迭代 M2T/编辑更新后的 block 状态。对位置 i 上的已提交 token x_i^old，当模型对该 token 的置信度在迭代间**下降**时触发重掩码：

$$\text{Remask}_{\text{LD}}(i) = \mathbb{1}\left[p_\theta^{(t-1)}(x_i^{\text{old}} \mid \mathbf{z}^{(t-1)}) - p_\theta^{(t)}(x_i^{\text{old}} \mid \mathbf{z}^{(t)}) > \tau_{\text{ld}}\right]$$

置信度下降说明：随着周围位置被逐步填充或修正，演化的上下文不再支持该 token——暗示它与正在收敛的邻域不兼容。反之，置信度上升意味着该 token 正在被演化的上下文验证，应当保留。与使用单步快照的 LowProb 不同，LogitDiff 捕捉跨迭代的*动态*上下文演化。每个 block 的第一次迭代因无前一步 logits 可对比，不触发重掩码。

### 3.3 安全机制

不受限的重掩码可能导致振荡（反复重掩码和重新预测同一位置）。我们采用两个保障：

**每位置重掩码预算：** 每个位置 i 维护一个计数器 c_i，每次被重掩码时递增。当 c_i ≥ C_max（默认 C_max = 5）时，重掩码被阻止，防止无限循环。

**重掩码比例上限：** 每步最多 ρ_max（默认 ρ_max = 0.25）比例的可编辑位置可被重掩码。当更多位置被标记时，我们优先处理置信度分数最低的（策略相关），确保最可疑的 token 优先被处理。

### 算法1：T2M 重掩码（替代 LLaDA2.1 中的 T2T 编辑）

```
输入：当前 block tokens z, 模型 p_θ, mask id [M], 策略 S, 阈值 τ, 预算 C_max, 比例上限 ρ_max

// 标准 M2T 步骤（与 LLaDA2.1 相同）
for 每个 mask 位置 i (z_i = [M]):
    if max_v p_θ(v | z) > τ_m2t:
        z_i ← argmax_v p_θ(v | z)

// T2M 重掩码步骤（替代 T2T 编辑）
E ← {i : z_i ≠ [M] 且 i ∉ prompt}          // 可编辑位置
R ← {i ∈ E : ShouldRemask_S(i, z, τ) 且 c_i < C_max}
if |R| > ρ_max · |E|:
    R ← 按最低置信度分数取 top-k, k = ⌊ρ_max · |E|⌋
for 每个 i ∈ R:
    z_i ← [M];  c_i ← c_i + 1
return R = ∅                                  // 若无重掩码则收敛
```

## 4. 理论分析

我们提供统一的理论框架来理解为什么 T2M 重掩码优于 T2T 编辑。分析沿四个互补轴展开。

### 4.1 解耦错误检测与纠正

T2T 编辑将两个逻辑上独立的决策耦合到单个条件中：(1) 当前 token 是否错误？(2) 是否存在置信的替代？这种耦合导致**纠正惰性 (correction inertia)**：尽管有明确证据表明 token 错误，系统仍无法启动纠正，因为没有单个替代候选达到置信阈值。我们通过**卡住集 (stuck set)** 形式化这一现象：

**定义（卡住集）：** 卡住集 S_stuck 包含当前 token 可能错误但没有单个替代超过置信阈值的位置：

$$\mathcal{S}_{\text{stuck}} = \left\{i : p_\theta(x_i^{\text{old}} \mid \mathbf{z}_{-i}) < \epsilon, \;\text{但}\; \max_v p_\theta(v \mid \mathbf{z}_{-i}) < \tau_{\text{t2t}}\right\}$$

这种情况在条件分布 p_θ(· | **z**_{-i}) 是多模态时自然出现。例如，补全"他感到非常____"：token "高兴"、"开心"、"愉快" 可能各获得约 0.25 的概率，而错误的已提交 token "紫色" 只有约 0.01 的概率。τ_t2t = 0.5 的 T2T 不会采取任何行动，因为没有单个候选超过阈值，尽管模型明显识别出了错误。

**命题1：** T2T 编辑无法纠正 S_stuck 中的任何位置。使用 LowProb 策略的 T2M 重掩码能纠正 S_stuck 中所有满足 p_θ(x_i^old | **z**_{-i}) < τ_lp 的位置。

**证明：** 对于 i ∈ S_stuck，T2T 要求 max_v p_θ(v | **z**_{-i}) > τ_t2t，这按定义失败。LowProb 只检查 p_θ(x_i^old | **z**_{-i}) < τ_lp。由于 S_stuck 中位置的 p_θ(x_i^old | **z**_{-i}) < ε ≪ τ_lp，这些位置被重掩码，随后在可能更收敛的上下文下被 M2T 步骤重新预测。∎

### 4.2 错误上下文净化

T2M 最关键的优势涉及重新预测时**上下文**的质量。在迭代去噪中，模型对位置 i 的预测以所有其他位置为条件：

$$p_\theta(x_i \mid \mathbf{z}_{-i}) = p_\theta\!\left(x_i \mid \underbrace{\mathbf{z}_{\text{correct}}}_{\text{正确 token}},\; \underbrace{\mathbf{z}_{\text{error}}}_{\text{错误 token}},\; \underbrace{\text{[M]}, \ldots, \text{[M]}}_{\text{未知}}\right)$$

**上下文中的错误 token。** 在并行生成中，错误不可避免：M2T 可能在 mask 位置填入错误 token，T2T 也可能将 token 替换为另一个同样错误的 token。这些错误 token——无论其来源——以合法 token 的形式出现在上下文中，提供误导性语义信号，偏置所有其他位置的预测。关键问题是：当检测到错误时，应如何处理该位置？

**T2T：替换为另一个 token。** T2T 将疑似错误替换为模型当前的最高预测。然而，即使该替换的置信度很高，模型做出这个预测时所依赖的上下文本身可能包含其他位置的错误 token——在受污染的上下文下，高置信度并不等于高正确率。因此替换本身可能也不正确，错误 token 仍以合法形式留在上下文中，不仅偏置其他位置的后续预测，也影响该位置自身在后续迭代中的重评估，形成自我强化的错误循环。

**T2M：重置为中性 mask。** T2M 用 [M] 替换可疑错误，模型将其识别为"未知"。Mask token 不携带语义方向——它完全中性，既不偏置其他位置的预测，也不干扰该位置自身后续的重预测。通过将检测到的错误逐步替换为 [M]，上下文中正确 token 与中性 mask 的比例提高，误导信号相应减少。

**注记 1（上下文信号层级）：** 对于预测位置 i，每个上下文位置 j ≠ i 提供三类信号之一：

1. **对齐信号**（z_j = x_j*，正确 token）：将 p_θ(x_i | **z**) 偏向正确的 x_i*，因为模型利用了正确的语义依赖关系。

2. **零信号**（z_j = [M]）：不将 p_θ(x_i | **z**) 偏向任何语义方向。模型在训练中大量接触 mask 上下文，已学会对未知位置产生校准良好的预测，有效地对该位置进行边缘化。

3. **对抗信号**（z_j = e_j ≠ x_j*，错误 token）：将 p_θ(x_i | **z**) 偏**离**正确的 x_i*。模型无法区分 e_j 和正确 token，因此信任错误的语义信号并相应调整预测。

在此层级下，T2M 将第 (3) 类信号替换为第 (2) 类，以零代价消除对抗性偏置源（零信号严格无害）。T2T 将第 (3) 类替换为一个新 token，该 token 本身可能仍是第 (3) 类，无法保证净改善。

**错误传播循环：** 当多个对抗信号共存时，它们相互作用：位置 j 的错误偏置位置 i 的预测，后者反过来在后续迭代中强化 j 处的错误。这种相互强化可能收敛到**稳定但不正确的不动点**。T2M 通过同时将多个对抗信号转换为零信号来打破这个循环，切断位置间的错误依赖关系。

### 4.3 训练-推理噪声不匹配

LLaDA2.1 采用双流训练目标：M2T 流训练模型从掩码上下文预测正确 token；T2T 流训练模型从随机扰动位置恢复 ground truth token（随机错误 token → ground truth）。因此，模型在训练时见过 [M] 噪声和随机 token 噪声两种类型。但训练噪声与推理错误之间存在关键的分布差异。

**随机扰动 ≠ 系统性错误。** T2T 训练中，扰动 token 是从词表中**均匀随机**采样的——与上下文无关、语义不连贯。然而在推理时，错误 token 是**系统性的、模型生成的**错误，通常语义上看似合理、局部连贯、且与周围 token 相关。这种差异**同时影响** M2T 和 T2T 操作：无论操作类型如何，推理时非 mask 上下文可能包含的模型生成错误，其分布与训练时见到的任何噪声都根本不同。

**Mask 作为严格更安全的上下文信号。** 鉴于这种不匹配，当检测到错误时最佳的做法是什么？Mask token 携带的误导信息严格少于错误 token：mask 表达"未知"，而错误 token 主动提供错误的语义信号。通过将疑似错误转为 [M]：

$$\{z_{\text{correct}}, z_{\text{error}}\} \xrightarrow{\text{T2M}} \{z_{\text{correct}}, \text{[M]}\}$$

T2M 将主动误导性信号替换为中性的"未知"标记。由于 M2T 从 [M] 上下文预测是掩码扩散模型的基础能力，这种转换利用了模型最可靠的运作模式。

**与连续扩散的类比。** 在连续扩散模型（如 DDPM）中，去噪 score function ∇_x log p(x) 用高斯噪声训练。如果测试时噪声分布偏离训练分布（如结构化的对抗性扰动而非随机高斯噪声），score 估计会退化。类似地，在掩码扩散中，模型用两种噪声类型训练：[M] token（M2T）和随机 token 扰动（T2T）。推理时的错误——系统性的、相关的、语义上看似合理的——构成了第三种未见过的噪声类型。T2M 通过将这种未见噪声转回熟悉的 [M] 类型来解决这一问题。

### 4.4 延迟提交与联合优化

T2T 是一种**贪心**策略：一旦识别到高置信替代，立即提交。这种贪心提交有两个问题：

1. **过早锁定。** 替换 token 一旦提交，就成为所有其他位置的上下文。如果替换本身次优（如多模态分布下的 argmax），错误不可逆地传播。

2. **逐位置独立。** T2T 独立优化每个位置：x̂_i = argmax p_θ(x_i | **z**_{-i})。这忽略了位置间依赖。

T2M 实现了**延迟提交**：通过将可疑位置重置为 [M]，将最终决策推迟到更多上下文收敛的后续步骤。当多个位置同时被重掩码时，后续 M2T 步骤有效地解决了**联合**优化：

$$\hat{\mathbf{x}}_{\mathcal{R}} = \arg\max_{\mathbf{x}_{\mathcal{R}}} \prod_{i \in \mathcal{R}} p_\theta(x_i \mid \mathbf{z}_{\text{correct}}, \text{[M]}^{|\mathcal{R}|})$$

其中 R 是重掩码位置集。虽然每个位置的预测在给定上下文下仍然独立，但上下文本身是**联合更干净的**（R 中的所有错误同时被移除），产生更好的个体预测，且更全局一致。

这类似于**模拟退火**：T2M 局部增加不确定性（重掩码）以跳出局部最优，探索更全局最优的解。

## 5. 相关工作

**离散掩码扩散语言模型。** 用于文本生成的掩码扩散模型快速发展。D3PM 引入了带吸收状态的离散扩散。MDLM 和 SEDD 开发了高效训练目标。LLaDA 扩展到 8B 参数并达到竞争性性能，LLaDA2.1 引入了 T2T 编辑以加速推理。Dream 进一步展示了扩散 LM 在规模上的可行性。

**掩码扩散中的重掩码。** ReMDM（NeurIPS 2025）是掩码扩散重掩码的开创性工作。它修改反向后验，引入*概率性*重掩码参数 σ_t，允许任何已 unmask 的 token 在每步被随机重掩码。ReMDM 在**采样过程级别**运作，修改整个反向扩散轨迹。相比之下，我们的方法专门在 LLaDA2.1 的**编辑步骤级别**运作，基于显式错误检测进行**定向**重掩码，而非随机重掩码。

利用注记 1 的信号层级框架，可以形式化地说明定向重掩码为何严格优于随机重掩码。考虑 N 个已提交（非 mask）位置，其中 N_c 个正确（对齐信号，价值 s_+ > 0），N_e 个错误（对抗信号，代价 s_- < 0）。定义上下文质量 Q = N_c · s_+ + N_e · s_-。**随机重掩码**以逐 token 概率 σ 独立重掩码每个位置，不区分正确与否：

$$Q_{\text{random}}(\sigma) = (1 - \sigma)(N_c \cdot s_+ + N_e \cdot s_-)$$

这产生了固有的两难：增大 σ 能移除对抗信号，但**同等程度地**破坏对齐信号。**定向重掩码**（假设完美检测）仅重掩码 N_e 个错误位置：

$$Q_{\text{targeted}} = N_c \cdot s_+$$

定向重掩码相对随机重掩码的优势为：

$$Q_{\text{targeted}} - Q_{\text{random}}(\sigma) = \underbrace{\sigma \cdot N_c \cdot s_+}_{\text{保留的对齐信号}} + \underbrace{(1-\sigma)(-N_e \cdot s_-)}_{\text{移除的对抗信号}} > 0$$

对所有 σ ∈ (0, 1] 严格成立。两项均为正：定向重掩码同时保留了更多正确上下文并移除了更多错误。即使检测不完美，只要检测器优于随机（即精确率-召回率权衡超过错误基准率 N_e / N），定向重掩码仍优于随机重掩码。

CORE（2026）提出上下文鲁棒重掩码，通过对掩码扰动的敏感度识别"上下文脆弱"token，认为静态基于概率的启发式是短视的。我们认为这一批评在理论上存在根本问题。CORE 的敏感度指标度量的是条件互信息 I(X_i; Z_S | Z_{-i,-S})，但该量受**当前去噪状态中 mask pattern 的支配**，而非 token 的正确性。具体而言：同一个正确 token，如果其语义相关的邻居尚未被预测（仍为 [M]），对上下文扰动的敏感度可能很低；一旦该邻居被填入（无论正确与否），敏感度可能骤变。反之，一个错误 token 也可能因为相关邻居恰好未被预测而表现"稳定"。换言之，CORE 的敏感度主要反映的是**去噪过程的瞬时状态**（哪些位置碰巧已被填充），而非 token 本身的正确性——这使其作为错误检测信号在理论上是不可靠的。

反之，静态概率 p_θ(x_i | **z**) 是模型在给定**完整**当前上下文下对 x_i 的**后验信念**——这是判断 token 正确性最直接的可用信号。由数据处理不等式，在人为退化的上下文（CORE 的掩码扰动）下评估，不可能获得比完整上下文预测严格更多的关于 X_i 正确性的信息。而且 CORE 的扰动分布（随机掩码上下文）本身是人为的，不反映推理时的实际错误模式。

实际层面，CORE 每次扰动测试需要 O(k) 次额外 forward pass，而静态概率是现有 forward pass 的免费副产品。我们的 LogitDiff 策略捕捉了一种相关但更本质的敏感度形式：它沿**实际**去噪轨迹跟踪模型意见的变化，而非在人为扰动下测试，同样零额外 forward pass 开销。

**基于训练的纠正。** 多种方法通过额外训练改进重掩码质量。RemeDi 训练了双流架构，通过 SFT + RL 学习逐 token 置信度分数。ProSeCo 添加 corrector 交叉熵损失，训练模型从自身错误中恢复。PRISM 附加轻量 adapter 并用自纠正损失微调。MDPO 将去噪建模为序列决策并应用策略优化，也提出了 Running Confidence Remasking (RCR) 作为 training-free baseline。这些方法与我们的方法互补：它们通过训练改进模型检测或纠正错误的能力，而我们改进的是**纠正机制本身**（T2M vs T2T），无需任何训练。

**定位。** 我们的方法沿三个维度提供了相对现有重掩码方法的原则性优势。**相比 ReMDM**：我们将盲目的随机重掩码替换为定向的错误驱动重掩码，并形式化证明其在上下文质量上严格优越（见 Q_random、Q_targeted 及其差值公式）。**相比 CORE**：我们避免了理论上不可靠的扰动敏感度指标（其值被瞬时 mask pattern 支配而非 token 正确性），转而使用直接的后验概率和自然去噪轨迹动态（LogitDiff），均基于模型的完整上下文预测。**相比训练方法**（RemeDi、ProSeCo、PRISM、MDPO）：我们在零训练开销下通过改进纠正机制本身（T2M vs T2T）实现提升，而非学习更好的错误检测器。我们的理论框架——信号层级（注记 1）、定向 vs 随机分析、延迟提交公式——提供了统一的理论基础，与上述所有方法互补且可组合。

## 6. 实验

### 6.1 实验设置

**模型。** 使用 LLaDA2.1-mini（16B 参数），Mixture-of-Experts 掩码扩散语言模型。所有实验使用相同预训练 checkpoint，无额外训练。

**推理超参数。** 采用 LLaDA2.1 推荐的 Quality Mode (Q Mode) 设置：M2T 置信阈值 τ_m2t = 0.7，T2T 编辑阈值 τ_t2t = 0.5，block 长度 B = 32，温度 = 0.0（greedy），eos_early_stop = True。生成长度按 benchmark 设定。

**Remask 超参数。** T2M 策略使用以下默认阈值（在 GSM-Plus 50 样本子集上调优）：LowProb τ_lp = 0.5，T2T-Remask τ_tr = 0.9，LogitDiff τ_ld = 0.3。安全参数：C_max = 5（每位置预算），ρ_max = 0.25（比例上限）。

**Baseline。** 主要对比原始 T2T 编辑（"Original"）和我们的三种 T2M 策略，所有方法使用相同模型和推理参数。这种受控对比隔离了编辑机制的效果。

**评测说明。** 我们使用 LLaDA2.1 官方 HuggingFace 模型权重、HuggingFace 提供的推理代码和官方 benchmark 数据集进行推理。然而，LLaDA2.1 团队未开源其评测代码，我们观察到在相同模型和数据集下，我们复现的 baseline 绝对分数与原始论文存在差异，这可能源于 prompt 模板或答案提取逻辑等评测细节的不同。这一差异不影响本工作的结论有效性：本文所有 T2T baseline 与 T2M 方法的比较均在**完全相同**的评测流程下进行，确保了公平性。

## 7. 结论

我们发现了 LLaDA2.1 中 Token-to-Token (T2T) 编辑的根本性局限，并提出 Token-to-Mask (T2M) 重掩码作为有原则的、training-free 的替代。通过将疑似错误 token 重置为 mask 状态而非用可能不正确的替代替换，T2M 净化了生成上下文、将系统性推理错误转回模型原生的 [M] 噪声类型、并通过延迟提交实现了联合多位置重优化。

我们的理论分析提供了统一的理解——为什么 mask token 严格优于错误 token 作为上下文，并在离散和连续扩散设置之间建立了类比。三种互补的错误检测策略（LowProb、T2T-Remask、LogitDiff）为检测不同类型的 token 级错误提供了灵活性。

**局限性：** 我们的方法特定于 LLaDA2.1 的 T2T 编辑机制，可能不直接适用于没有此机制的 dLLM。理论分析依赖于 mask 作为上下文信号优于错误 token 的假设，当错误率极低且 T2T 训练充分覆盖推理错误模式时可能不成立。安全机制引入了可能需要任务特定调优的超参数（C_max, ρ_max）。

**未来工作：** 本工作的核心发现之一是*训练-推理噪声不匹配*：T2T 训练中使用的随机 token 扰动与推理时遇到的系统性、模型生成的错误存在根本差异。虽然 T2M 通过将错误重置为模型原生的 [M] 噪声类型来缓解这一问题，但更直接的方案是从训练层面弥合这一差距。具体而言，*噪声感知的继续训练*（noise-aware continued training）可以将 T2T 训练流中的随机扰动替换为真实的模型生成错误——通过运行模型自身的推理流程并记录实际产生的错误 token 来收集。在这种"真实噪声"上训练，将使模型暴露于推理时将面对的系统性错误模式，有望同时改善被污染上下文下的 M2T 预测和 T2T 纠正准确性。这与我们的推理时方法互补：T2M 改进纠正*机制*，而噪声感知训练提升模型对真实错误的*鲁棒性*。

此外，将 T2M 重掩码与采样级重掩码（如 ReMDM）结合可能带来进一步提升。自适应学习重掩码阈值（如通过 PRISM 框架）而非使用固定启发式是一个有前景的方向。将分析扩展到 LLaDA2.1 之外的其他 dLLM 架构也值得探索。
